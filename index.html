Actividad 3: Callbacks

Definición: Un callback es una función que se pasa como argumento a otra función para ser ejecutada después de un proceso.

Actividad 4: Promises

Definición: Objeto que representa la terminación o falla de una operación asíncrona.

Actividad 5: Async / Await

Definición: Permite manejar promesas con una sintaxis que parece síncrona, mejorando la legibilidad.

Actividad 6: Destructuring Objects

Definición: Sintaxis que permite desempacar valores de arreglos o propiedades de objetos en variables distintas.
